Prerequisites
  • Windows 10/11 Pro or Enterprise (for Hyper-V) — or Windows Home with WSL2
  • At least 8 GB RAM (16 GB recommended)
  • At least 10 GB free disk space
  • Internet connection (for downloading Docker images)
  • The MXSuite source code folder (D:\growthzone\)
ℹ️
You do NOT need Java, Node.js, or Maven installed. Docker handles all the building.
1
Install Docker Desktop
~10 minutes

1a. Download Docker Desktop

Go to docker.com/products/docker-desktop and download the Windows installer.

1b. Run the Installer

  • Double-click Docker Desktop Installer.exe
  • When prompted, select "Use WSL 2 instead of Hyper-V" (recommended)
  • Click OK and wait for installation to complete
  • Restart your computer when prompted

1c. Start Docker Desktop

  • After restart, Docker Desktop should start automatically
  • Accept the license agreement
  • Wait for the Docker Engine to start (the whale icon in the system tray will stop animating)
  • You can skip the sign-in / tutorial steps

1d. Verify Docker is Working

Open PowerShell (search for "PowerShell" in the Start menu) and run:

docker --version
docker compose version

You should see version numbers. If you get an error, Docker Desktop may still be starting up — wait a minute and try again.

⚠️
Windows Home users: You need WSL 2 installed. If the Docker installer asks you to install it, follow the prompts. You may need to run wsl --install in PowerShell (as Administrator) first.
2
Configure Environment
~2 minutes

2a. Open the Project Directory

Open PowerShell and navigate to the project:

cd D:\growthzone

2b. Create the .env File

Copy the example configuration:

Copy-Item .env.example .env

2c. Edit the .env File

Open .env in Notepad or any text editor:

notepad .env

For local development, the dev overlay (docker-compose.dev.yml) provides safe defaults for passwords and secrets. But you should review these key values:

VariableDev DefaultNotes
POSTGRES_PASSWORDmxsuite_devRequired in production (no default)
JWT_SECRET(long dev string)Required in production (no default)
MXSUITE_ADMIN_EMAILadmin@mxsuite.comYour admin login email
MXSUITE_ADMIN_PASSWORDAdmin123!Required in production (no default)
FRONTEND_PORT80Change to 8088 if port 80 is in use
⚠️
Production mode (.\deploy.ps1 -Prod) has no default passwords. You must set POSTGRES_PASSWORD, JWT_SECRET, and MXSUITE_ADMIN_PASSWORD in your .env file or the services will fail to start.
ℹ️
The bootstrap admin account is only created on the very first startup when the database is empty. Changing these values later has no effect — use the admin UI to manage users after that.
3
Build and Start MXSuite
~10-15 minutes (first time)

Option A: Use the Deploy Script (Recommended)

The deploy script handles everything automatically:

cd D:\growthzone
.\deploy.ps1
⚠️
If you get a "scripts are disabled" error, run this first:
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

By default, deploy.ps1 runs in development mode, which merges docker-compose.yml + docker-compose.dev.yml. This exposes database/Redis ports, enables Java remote debugging, and starts MailHog for email testing.

The script will:

  1. Verify Docker is running
  2. Create .env from .env.example if it doesn't exist
  3. Build Docker images (Java backend + all React frontends via pnpm)
  4. Start all services (PostgreSQL, Redis, API, Frontend, MailHog)
  5. Wait for health checks to pass
  6. Print the access URLs (including MailHog and debug port)

For production mode (no exposed ports, no debug, no MailHog):

.\deploy.ps1 -Prod

Option B: Manual Docker Commands

If you prefer running commands directly:

# Development mode (recommended for local)
cd D:\growthzone
docker compose -f docker-compose.yml -f docker-compose.dev.yml build
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d

# Production mode (no dev services)
docker compose build
docker compose up -d

# Watch the logs while it starts up
docker compose logs -f

What Happens During First Build

Docker will:

  1. Download base images (Java 21, Node 22, PostgreSQL 16, Redis 7, Nginx)
  2. Build the API: install Maven parent POM → download Java dependencies → compile 56 source files → package JAR
  3. Build 6 frontend apps: pnpm workspace install → TypeScript compile → Vite bundle (shared lib first, then MFEs in parallel)
  4. Create production images with only the compiled artifacts
ℹ️
Subsequent builds are much faster (~1-2 minutes) because Docker caches the dependency download layers. Only changed source code triggers a rebuild.

5
Daily Operations

Starting MXSuite (after a reboot)

  1. Make sure Docker Desktop is running (check the whale icon in system tray)
  2. Open PowerShell and run:
cd D:\growthzone
.\deploy.ps1

If images are already built, services start in ~5 seconds.

Stopping MXSuite

.\deploy.ps1 -Down

Your data is preserved in Docker volumes. It will be there when you start again.

Viewing Logs

# All services
.\deploy.ps1 -Logs

# Just the API
docker compose -f docker-compose.yml -f docker-compose.dev.yml logs -f mxsuite-api

# Just the frontend
docker compose -f docker-compose.yml -f docker-compose.dev.yml logs -f mxsuite-frontend

Rebuilding After Code Changes

# Quick rebuild (uses cache)
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build

# Full rebuild (no cache)
.\deploy.ps1 -Rebuild

Testing Emails

In dev mode, all outgoing emails are caught by MailHog — no emails are actually sent. Open the MailHog UI to view them:

http://localhost:8025

This captures invitation emails, password reset emails, and any other notifications.

Remote Debugging (Java)

In dev mode, the API exposes JDWP on port 5005. Connect your IDE's remote debugger to localhost:5005.

Resetting Everything (Fresh Start)

⚠️
This deletes all data — database, uploaded files, Redis cache. Use only when you want a completely fresh start.
.\deploy.ps1 -Reset

Troubleshooting

Port 80 is already in use

Another application (IIS, Apache, Skype) is using port 80. Either stop that service, or change the port:

  1. Edit .env and set FRONTEND_PORT=8088
  2. Restart: .\deploy.ps1
  3. Access at http://localhost:8088

Port 5432 is already in use

You may have a local PostgreSQL installation. Either stop it, or change the port:

  1. Edit .env and set POSTGRES_PORT_HOST=5433
  2. Restart: .\deploy.ps1

Build fails with "out of memory"

Docker Desktop may need more RAM. Open Docker Desktop → Settings → Resources:

  • Set Memory to at least 4 GB (6 GB recommended)
  • Set CPUs to at least 2

Build fails with pnpm/npm errors

Try a full clean rebuild:

.\deploy.ps1 -Rebuild

If you see ERR_PNPM_FROZEN_LOCKFILE, ensure pnpm-lock.yaml exists in the project root. Generate it by running pnpm install locally (requires Node.js + pnpm installed on your machine).

API container keeps restarting

Check the logs for the specific error:

.\deploy.ps1 -Logs

Common causes:

  • "Connection refused" to postgres — PostgreSQL isn't ready yet. Wait 30 seconds and check again. The API has a health check with start period of 60 seconds.
  • "Flyway migration failed" — Database schema issue. Try .\deploy.ps1 -Reset for a fresh start.
  • "JWT secret" error — Your JWT_SECRET in .env may be too short. Use at least 32 characters.

Frontend shows "Failed to fetch remote" errors

The micro-frontend assets may not have built correctly. Rebuild the frontend image:

.\deploy.ps1 -Rebuild

Cannot log in (401 Unauthorized)

  • If this is the first startup, the admin account is created from .env values. Check MXSUITE_ADMIN_EMAIL and MXSUITE_ADMIN_PASSWORD.
  • If you changed the password in .env after the first startup, the change has no effect. Use the password reset flow or connect to the database directly.

Docker Desktop won't start

  • Windows Home: Ensure WSL 2 is installed: wsl --install (in Admin PowerShell)
  • Windows Pro: Ensure Hyper-V is enabled in Windows Features
  • Check that virtualization is enabled in BIOS/UEFI

Everything is broken — fresh start

# Nuclear reset via deploy script
.\deploy.ps1 -Reset

# Or manually:
docker compose down -v
docker rmi mxsuite-api mxsuite-frontend 2>$null
.\deploy.ps1 -Rebuild
